# Описание файлов в корне проекта

## Необходимые для запуска пайплайнов

### `pyproject.toml`
**Назначение:** Конфигурация проекта Python: метаданные, зависимости, настройки инструментов (ruff, mypy, pytest, black). Определяет точку входа CLI (`bioetl = "bioetl.cli.main:app"`).  
**Необходимость:** ✅ **КРИТИЧНО** — требуется для установки пакета (`pip install -e ".[dev]"`) и работы CLI.

### `requirements.txt`
**Назначение:** Зафиксированные версии зависимостей проекта (pandas, pandera, requests и др.).  
**Необходимость:** ✅ **ОБЯЗАТЕЛЬНО** — используется для установки зависимостей при развёртывании.

### `sitecustomize.py`
**Назначение:** Добавляет `src/` в `sys.path` для корректной работы CLI entrypoints.  
**Необходимость:** ✅ **ОБЯЗАТЕЛЬНО** — требуется для корректного разрешения импортов при запуске CLI команд.

## Документация

### `README.md`
**Назначение:** Главная документация проекта: описание, быстрый старт, поддерживаемые источники данных, ссылки на документацию.  
**Необходимость:** ⚠️ **ОПЦИОНАЛЬНО** — важна для понимания проекта, но не требуется для запуска.

### `CHANGELOG.md`
**Назначение:** История изменений проекта: breaking changes, новые функции, изменения, улучшения качества.  
**Необходимость:** ⚠️ **ОПЦИОНАЛЬНО** — для отслеживания изменений, не влияет на запуск.

### `DEPRECATIONS.md`
**Назначение:** Реестр устаревших символов и планов их удаления.  
**Необходимость:** ⚠️ **ОПЦИОНАЛЬНО** — для разработчиков, не требуется для запуска.


## Файлы, перемещённые в `tools/`

## Минимальный набор для запуска пайплайнов

Для запуска пайплайнов **обязательно** наличие:
1. `pyproject.toml` — конфигурация проекта
2. `requirements.txt` — зависимости
3. `sitecustomize.py` — настройка путей импорта
4. Структура `src/bioetl/` — код проекта
5. Конфигурации пайплайнов в `src/bioetl/configs/pipelines/`

Остальные файлы (README.md, CHANGELOG.md, DEPRECATIONS.md) улучшают процесс разработки и документирования, но не требуются для выполнения пайплайнов. Все опциональные файлы перенесены в `tools/`.
